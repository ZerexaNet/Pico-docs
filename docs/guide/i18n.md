# i18n 翻译配置

DocFlow 的 i18n 配置在 `docs.config.mjs` 的 `i18n` 字段中。

## 示例

```js
i18n: {
  enabled: true,
  endpoint: "/api/translate",
  upstreamEndpoint: "https://deepl.io.hk.cn/translate",
  sourceLang: "zh",
  defaultLang: "zh",
  altCount: 2,
  cache: true,
  autoApplySaved: true,
  languages: [
    { code: "zh", label: "简体中文" },
    { code: "en", label: "English" }
  ]
}
```

## 字段说明

- `enabled`：是否启用 i18n 功能
- `endpoint`：前端调用的翻译接口地址
- `upstreamEndpoint`：当 `endpoint` 使用本地代理路径时，上游翻译服务地址
- `sourceLang`：源语言
- `defaultLang`：默认语言
- `altCount`：候选翻译数量（0-3）
- `cache`：是否缓存翻译结果
- `autoApplySaved`：是否自动应用用户上次选择的语言

## endpoint 和 upstreamEndpoint 的关系

- 当 `endpoint` 为 `/api/translate` 这类本地路径时，开发服务器会把请求转发到 `upstreamEndpoint`
- 当 `endpoint` 是 `https://...` 这类外部完整 URL 时，前端会直接请求该地址

## 接口请求体

翻译请求使用 `POST`，JSON 字段如下：

- `text`（必填）
- `source_lang`（可选）
- `target_lang`（必填）
- `alt_count`（可选）
